## 二、阵列间距设计（28GHz 毫米波）

### 2.1 波长计算

$$\lambda = \frac{c}{f} = \frac{3 \times 10^8}{28 \times 10^9} \approx 1.071 \text{ cm}$$

### 2.2 统一间距标准

为完全避免栅瓣、保证波束定向能力，三类阵列统一采用 **0.5λ** 间距：

$$d = 0.5\lambda \approx 0.536 \text{ cm} = 0.00536 \text{ m}$$

### 2.3 三类阵列间距汇总

| 阵列 | 类型 | 阵元数 | 间距 d | 阵列总尺寸 | 代码位置 |
|------|------|--------|--------|-----------|---------|
| BS | ULA 线阵 | 8 | 0.5λ ≈ 0.536cm | (8-1)×d ≈ **3.75cm** | channel.py 数学模型 |
| FAS | ULA 线阵 | 16 | 0.5λ ≈ 0.536cm | (16-1)×d ≈ **8.04cm** | entity.py 端口坐标 |
| RIS | UPA 面阵 | 8×8=64 | 0.5λ ≈ 0.536cm | 7×d ≈ **3.75cm** × 3.75cm | entity.py RIS 坐标 |

### 2.4 无人机机身容纳性

FAS 16 端口线阵总长度 ≈ 8.04 cm，无人机机身横向空间完全可以容纳，尺寸合理。

### 2.5 信道模型中的间距体现

信道模型（`channel.py`）中的阵列响应向量隐含了 0.5λ 间距假设：

```python
# ULA 阵列响应 (channel.py:149)
# a[i] = exp(j × sin(θ) × cos(φ) × i × π)
# 其中 i×π 对应 0.5λ 间距的相位差
Linear_response[i, 0] = cmath.exp(1j * math.sin(theta) * math.cos(fai) * i * math.pi)
```

当 d = 0.5λ 时，相邻阵元相位差 = 2π × d / λ = π，与代码一致。

## 三、FAS 流体天线（Fluid Antenna System）

### 3.1 定义与类型

**类型：波束可重构离散端口流体天线（Beam-Reconfigurable Discrete-Port FAS）**

FAS 是一种基于**液态金属微流控技术**的新型天线系统。与传统相控阵天线不同，FAS 不使用移相器，而是通过在天线表面上**切换不同的物理端口**来改变有效辐射位置，从而实现波束指向的调整。

**代码定义**（`entity.py:444-492`）：

```python
class UAV_BS_FAS(object):
    """
    波束可重构离散端口流体天线:
    - UAV mobility platform (无人机移动平台)
    - Base Station (BS) active beamforming (基站主动波束成形)
    - FAS: 波束可重构离散端口流体天线，通过端口切换改变波束指向

    核心原理：有限离散空间端口切换，改变波束指向，实现对准RIS、零陷窃听者
    """
```

### 3.2 物理结构

**端口排布**：16 个离散端口在无人机机身上**水平线性排布**，间距 0.5λ ≈ 0.536cm。

**阵列间距设计依据**（28GHz 毫米波）：
- 波长：λ = c/f = 3×10⁸ / 28×10⁹ ≈ 1.071 cm
- 阵元间距：d = 0.5λ ≈ 0.536 cm = 0.00536 m
- 16 端口线阵总长度：(16-1) × 0.00536 ≈ 8.04 cm（无人机机身可容纳）

### 3.3 与传统相控阵天线的对比

| 特性 | 传统相控阵天线 | FAS 流体天线 |
|------|--------------|-------------|
| **波束控制方式** | 每个天线元素配备独立移相器，通过调整相位实现波束扫描 | 切换不同的物理端口，改变等效辐射中心位置 |
| **硬件复杂度** | N 个天线需要 N 个移相器 + N 条射频链 | 1 个射频链 + 1 个端口切换开关 |
| **波束调整范围** | 连续可调（360°） | 离散切换（16 个端口 = 16 个波束方向） |
| **物理原理** | 电子波束成形（相位干涉） | 空间端口切换（辐射位置偏移） |
| **功耗** | 较高（移相器阵列） | 较低（仅端口开关） |
| **适用场景** | 需要快速扫描的雷达/通信 | 无人机等功耗受限平台 |

### 3.4 工作原理

**核心思想**：激活不同的端口 → 改变天线的有效辐射中心 → 波束指向随之改变。

**端口切换的物理效应**：

```
激活端口 [-8]（最左端）：
    有效辐射中心偏左 → 波束指向偏右 → 信号偏向 RIS 方向

    ┌──────────────────────────────────────┐
    │  [█] [ ] [ ] [ ] [ ] [ ] [ ] [ ] ... │  ← 仅端口-8激活
    │   ●                                    │
    │         ╲                              │
    │           ╲  波束方向                   │
    │             ╲                          │
    │               → RIS                    │
    └──────────────────────────────────────┘

激活端口 [0]（中心）：
    有效辐射中心在中央 → 波束正前方

    ┌──────────────────────────────────────┐
    │  [ ] [ ] [ ] [ ] [ ] [ ] [ ] [█] ... │  ← 仅端口0激活
    │                  ●                     │
    │                  ↓                     │
    │              波束正前方                 │
    └──────────────────────────────────────┘

激活端口 [7]（最右端）：
    有效辐射中心偏右 → 波束指向偏左

    ┌──────────────────────────────────────┐
    │  [ ] [ ] [ ] [ ] [ ] [ ] [ ] [ ]...[█]│  ← 仅端口7激活
    │                                  ●     │
    │                              ╱         │
    │                          ╱             │
    │                      ╱                 │
    │               波束方向                  │
    └──────────────────────────────────────┘
```

**实际使用中**：RL agent 不是选择单一端口，而是输出一个 16 维的权重向量 `F`，同时控制所有端口的信号权重：

```python
# FAS 发射波束成形矩阵 F (16×1)
# 每个元素对应一个端口的信号权重（复数）
F = [f₁, f₂, ..., f₁₆]ᵀ

# 激活多个端口时，有效辐射中心 = 加权平均位置
# 有效辐射中心 = Σ(|fᵢ|² × position_i) / Σ|fᵢ|²
```

### 3.5 FAS 在信道模型中的体现

**FAS 增益计算**（`env_uav_bs_fas.py`）：

```python
# 计算 FAS 平均增益（标量）
fas_gain = np.mean(np.abs(self.UAV_BS_FAS.F))
# F 是 RL agent 输出的 16×1 复数向量
# |F| 取每个端口的幅度，mean 取平均
```

**FAS 增益的物理含义**：
- `F` 的每个元素控制对应端口的信号权重
- `mean(|F|)` 反映了所有端口的平均发射功率
- 当多个端口协同工作时，有效辐射中心位于加权平均位置
- FAS 增益作为标量因子乘到信道上：`H_eff = H_channel × fas_gain`

**FAS 动作空间**（`main_train_uav_bs_fas.py:228`）：

```python
# RL agent 输出 Phi 向量，前 32 维控制 FAS
# 每个端口需要 2 个值（实部 + 虚部），共 16 端口 × 2 = 32 维
self.UAV_BS_FAS.F = convert_list_to_complex_matrix(
    Phi[:self.UAV_BS_FAS.fas_ant_num*2],    # 取前 32 维
    (self.UAV_BS_FAS.fas_ant_num, 1)         # 形状 (16, 1)
) * math.pow(self.power_factor, 0.5)         # × √100 功率缩放
```

### 3.6 FAS 端口优化算法

**`fas_beam_optimize` 方法**（`entity.py:523-546`）：

```python
def fas_beam_optimize(self, ris_pos, eaves_pos):
    """
    遍历所有 FAS 端口，自动选最优端口：
    最大化 FAS→RIS 增益，同时最小化 FAS→窃听者 增益

    优化目标：metric = gain_ris - gain_eve
    """
    for idx in range(self.fas_num_ports):
        g_ris, _ = self.get_channel_gain(idx, ris_pos)  # 端口→RIS 增益
        g_eve, _ = self.get_channel_gain(idx, eaves_pos) # 端口→窃听者 增益
        metric = g_ris - g_eve  # 差值最大 = 最优端口
```

**物理意义**：选择一个端口，使得发射信号在 RIS 方向的增益最大，同时在窃听者方向的增益最小（天然零陷）。

## 四、RIS 可重构智能反射面（Reconfigurable Intelligent Surface）

### 4.1 定义与类型

**类型：多功能有源可重构智能反射面（Multi-functional Active RIS）**

这是一个**有源 RIS**（Active RIS），而非传统的无源 RIS。有源 RIS 的核心特点是：
- 每个反射单元不仅调整相位，还能**放大信号幅度**
- 可以**独立生成人工噪声**干扰窃听者
- 需要外部供电，有功耗约束

**代码定义**（`entity.py:66-115`）：

```python
class RIS(object):
    """
    多功能可重构智能反射面 (Multi-functional RIS):
    1. 信号反射：将基站信号反射到目标用户，支持幅度放大
    2. 干扰生成：生成人工噪声干扰窃听者
    3. 动态调度：根据场景需求分配RIS元素用途

    工作模式:
    - MR (Multi-functional RIS): 同时反射和干扰  ← 当前使用
    - AR (Active RIS): 仅反射（带放大）
    - AJ (Active Jamming): 仅干扰
    """
```

### 4.2 有源 RIS vs 无源 RIS

| 特性 | 无源 RIS (Passive) | 有源 RIS (Active) | 本模型 |
|------|-------------------|------------------|--------|
| **幅度控制** | \|Φ_nn\| = 1（纯相移） | \|Φ_nn\| ≥ 1（可放大） | **有源** ✅ |
| **信号放大** | 无，仅反射 | 有，最大放大 10 倍 | **有放大** ✅ |
| **人工噪声** | 不能生成 | 可独立生成 AN | **有 AN** ✅ |
| **功耗** | 极低（无源器件） | 较高（需要放大电路） | 有功率约束 |
| **硬件成本** | 低 | 高（需要放大器） | - |
| **干扰能力** | 仅被动反射 | 可主动发射干扰信号 | **主动干扰** ✅ |

### 4.5 相位矩阵体系

RIS 维护三套独立的对角相位矩阵：

```python
# entity.py:107-109 — RIS 64单元时
self.Phi     = diag(ones(64))    # 总相位矩阵（单位矩阵初始化）
self.theta_R = diag(ones(64))    # 反射相位矩阵（控制信号反射）
self.theta_J = diag(zeros(64))   # 干扰相位矩阵（控制人工噪声）
```

```
三套矩阵的关系：

Phi (总矩阵) = theta_R (反射) + theta_J (干扰)

Phi     ──→  信号反射路径（增强用户信号）
theta_R ──→  反射分量（与 Phi 相关）
theta_J ──→  干扰分量（生成 AN 干扰窃听者）
```

### 4.6 相位映射（convert_list_to_complex_diag）

**RL agent 输出的 action 值 → RIS 复数对角元素**（`math_tool.py:122-136`）：

```python
def convert_list_to_complex_diag(list_real, diag_row_num):
    for i in range(M):
        # 幅度映射：action ∈ [-1, 1] → amplitude ∈ [1, 10]
        amplitude = 1.0 + 9.0 * (0.5 + 0.5 * math.sin(list_real[i]))
        # 相位映射：action ∈ [-1, 1] → phase ∈ [-π, π]
        phase = list_real[i] * math.pi
        # 复数对角元素 = 幅度 × e^(j×相位)
        diag_matrix_complex[i, i] = amplitude * cmath.exp(1j * phase)
```

**映射关系图**：

```
action ∈ [-1, 1]
    │
    ├──→ amplitude = 1 + 9 × (0.5 + 0.5 × sin(action))
    │    范围: [1, 10]  ← 有源放大，最大10倍
    │
    └──→ phase = action × π
         范围: [-π, π]  ← 全相位覆盖

最终: Phi_nn = amplitude × e^(j × phase)   (复数对角元素)
```

### 4.7 双路径工作模式（当前模型）

**窃听者接收信号模型**：

```
y_e = [直射路径] + [RIS反射路径] + [人工噪声] + n_e
    = h_d^H × G × s_k + h_R_e^H × Phi_signal × h_UR × G × s_k
      + h_R_e^H × Phi_jam × z_AN + n_e
         ↑                    ↑                       ↑
    UAV→窃听者 直射    UAV→RIS→窃听者 反射      RIS→窃听者 AN
```

**信道维度**：

| 信道 | 符号 | 维度 | 路径损耗指数 |
|------|------|------|------------|
| UAV→RIS | $h_{UR}$ | (64, 8) | n = 2.2 |
| RIS→用户k | $h_{R,k}$ | (1, 64) | n = 2.8 |
| RIS→窃听者 | $h_{R,e}$ | (1, 64) | n = 2.8 |

## 五、FAS 与 RIS 的协同工作

### 5.1 协同机制

```
FAS (无人机上)          RIS (地面固定)
    │                      │
    │  ① FAS 选择最优端口    │
    │  对准 RIS 方向发射     │
    │ ──────────────────→  │
    │                      │  ② RIS Phi_signal 反射信号
    │                      │  增强用户接收
    │                      │ ────────────→ 用户1 ✓
    │                      │ ────────────→ 用户2 ✓
    │                      │
    │                      │  ③ RIS Phi_jam 生成 AN
    │                      │  干扰窃听者
    │                      │ ═══════════→ 窃听者 ✗
```

### 5.2 优化目标

| 组件 | 优化目标 | 实现方式 |
|------|---------|---------|
| FAS | 最大化→RIS增益，最小化→窃听者增益 | 端口切换改变波束指向 |
| RIS Phi_signal | 增强用户信号（相长干涉） | 调整反射相位对齐直射路径 |
| RIS Phi_jam | 干扰窃听者（人工噪声） | 生成 AN + 相位对齐窃听者信道 |
| BS G | 波束成形，多用户干扰管理 | 空间复用 + 干扰抑制 |

### 5.3 功率分配

```
总功率 = BS发射功率 + FAS发射功率 + RIS放大功率 + AN功率

BS:     P_BS = trace(G × G^H) × power_factor    (power_factor = 100)
FAS:    P_FAS = mean(|F|)² × power_factor
RIS:    P_RIS ≤ Pr = 0 dBm = 1 mW              (总功率约束)
AN:     P_AN = P_J = 10 dBm = 10 mW            (干扰功率)
```
