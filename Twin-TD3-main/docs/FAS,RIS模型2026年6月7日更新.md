## 二、阵列间距设计（28GHz 毫米波）

### 2.1 波长计算

$$\lambda = \frac{c}{f} = \frac{3 \times 10^8}{28 \times 10^9} \approx 1.071 \text{ cm}$$

### 2.2 统一间距标准

三类阵列统一采用 **0.5λ** 间距：

$$d = 0.5\lambda \approx 0.536 \text{ cm} = 0.00536 \text{ m}$$

### 2.3 三类阵列间距汇总

| 阵列 | 类型 | 阵元数 | 间距 d | 阵列总尺寸 | 代码位置 |
|------|------|--------|--------|-----------|---------|
| BS | ULA 线阵 | **4** | 0.5λ ≈ 0.536cm | (4-1)×d ≈ **1.61cm** | src/envs/entity.py |
| FAS | ULA 线阵 | **12** | 0.5λ ≈ 0.536cm | (12-1)×d ≈ **5.90cm** | src/envs/entity.py |
| RIS | UPA 面阵 | 8×8=64 | 0.5λ ≈ 0.536cm | 7×d ≈ **3.75cm** × 3.75cm | src/envs/entity.py |

### 2.4 信道模型中的间距体现

```python
# channel.py — ULA 阵列响应
# a[i] = exp(j × sin(θ) × cos(φ) × i × π)
# 其中 i×π 对应 0.5λ 间距的相位差
Linear_response[i, 0] = cmath.exp(1j * math.sin(theta) * math.cos(fai) * i * math.pi)
```

## 三、FAS 流体天线（Fluid Antenna System）

### 3.1 定义与类型

**类型：波束可重构离散端口流体天线（Beam-Reconfigurable Discrete-Port FAS）**

FAS 是一种基于**液态金属微流控技术**的新型天线系统。通过在天线表面上**切换不同的物理端口**来改变有效辐射位置，从而实现波束指向的调整。

**代码定义**（`src/envs/entity.py`）：

```python
class UAV_BS_FAS(object):
    """
    UAV-BS-FAS Integrated Entity (波束可重构离散端口流体天线):
    - UAV mobility platform (无人机移动平台)
    - Base Station (BS) active beamforming (基站主动波束成形)
    - FAS: 波束可重构离散端口流体天线，通过端口切换改变波束指向

    无人机搭载设备：
    - 基站(BS)：4根天线(ULA)，用于主动波束成形
    - FAS流体天线：12个离散端口(ULA)，实际使用4个关键端口

    核心原理：有限离散空间端口切换，改变波束指向，实现对准RIS、零陷窃听者
    """
```

### 3.2 物理结构

**端口排布**：12 个离散端口在无人机机身上**水平线性排布**，间距 0.5λ ≈ 0.536cm。

```python
# src/envs/entity.py — 端口局部坐标定义
self.fas_port_spacing = 0.5 * 3e8 / 28e9  # 0.5λ = 0.00536m
self.fas_port_local = np.array([
    [self.fas_port_spacing * i, 0.0, 0.0] for i in range(-fas_num_ports // 2, fas_num_ports // 2)
])
```

**端口坐标映射**（以 12 端口为例，间距 0.00536m）：

```

局部坐标: -0.0322 -0.0268 -0.0214 -0.0161 -0.0107 -0.0054  0.0000  0.0054  0.0107  0.0161  0.0214  0.0268  (m)
           ←──────────────── 无人机机身 (水平方向, 总长5.90cm) ────────────────→
```

### 3.3 与传统相控阵天线的对比

| 特性 | 传统相控阵天线 | FAS 流体天线 |
|------|--------------|-------------|
| **波束控制方式** | 每个天线元素配备独立移相器 | 切换不同的物理端口 |
| **硬件复杂度** | N 个天线需要 N 个移相器 + N 条射频链 | 1 个射频链 + 1 个端口切换开关 |
| **波束调整范围** | 连续可调（360°） | 离散切换（12 个端口 = 12 个波束方向） |
| **物理原理** | 电子波束成形（相位干涉） | 空间端口切换（辐射位置偏移） |
| **功耗** | 较高（移相器阵列） | 较低（仅端口开关） |
| **适用场景** | 需要快速扫描的雷达/通信 | 无人机等功耗受限平台 |

### 3.4 工作原理

**核心思想**：激活不同的端口 → 改变天线的有效辐射中心 → 波束指向随之改变。

**实际使用中**：RL agent 输出一个 8 维的权重向量 `F`（4 个关键端口 × 2 实虚），同时控制所有端口的信号权重：

```python
# src/envs/uav_comm_env.py — 实际 FAS 波束成形
fas_beam_action = G[16:24]  # 8维: FAS波束 (4端口×2实虚)
self.UAV_BS_FAS.F = convert_list_to_complex_matrix(
    fas_beam_action, (4, 1)  # 形状 (4, 1)
) * math.pow(self.power_factor, 0.5)
```

### 3.5 FAS 端口优化算法

**`fas_beam_optimize` 方法**（`src/envs/entity.py`）：

```python
def fas_beam_optimize(self, ris_pos, eaves_pos):
    """
    遍历所有 FAS 端口，按 metric = g_ris - g_eve 排序返回候选列表
    实现：对准RIS传输 + 天然零陷干扰窃听者
    """
    for idx in range(self.fas_num_ports):
        g_ris, _ = self.get_channel_gain(idx, ris_pos)
        g_eve, _ = self.get_channel_gain(idx, eaves_pos)
        metric = g_ris - g_eve
```

**`fas_port_select_with_rl` 方法**（`src/envs/entity.py`）：

```python
def fas_port_select_with_rl(self, rl_signal, ris_pos, eaves_pos):
    """
    启发式 + RL微调的FAS端口选择
    - rl_signal < -0.3: 选候选第2名 (探索非最优端口)
    - -0.3 ≤ rl_signal ≤ 0.3: 选启发式最优 (保持稳定)
    - rl_signal > 0.3: 选候选第3名 (进一步探索)
    """
```

### 3.6 FAS 在信道模型中的体现

**FAS 增益计算**（`src/envs/uav_comm_env.py`）：

```python
# 计算 FAS 平均增益（标量）
fas_gain = np.mean(np.abs(self.UAV_BS_FAS.F))
# F 是 RL agent 输出的 4×1 复数向量
# |F| 取每个端口的幅度，mean 取平均
```

**FAS 增益的物理含义**：
- `F` 的每个元素控制对应端口的信号权重
- `mean(|F|)` 反映了所有端口的平均发射功率
- 当多个端口协同工作时，有效辐射中心位于加权平均位置
- FAS 增益作为标量因子乘到信道上：`H_eff = H_channel × fas_gain`

## 四、RIS 可重构智能反射面（Reconfigurable Intelligent Surface）

### 4.1 定义与类型

**类型：多功能有源可重构智能反射面（Multi-functional Active RIS）**

这是一个**有源 RIS**（Active RIS），核心特点：
- 每个反射单元不仅调整相位，还能**放大信号幅度**
- 可以**独立生成人工噪声**干扰窃听者
- 需要外部供电，有功耗约束

**代码定义**（`src/envs/entity.py`）：

```python
class RIS(object):
    """
    多功能可重构智能反射面 (Multi-functional RIS):
    1. 信号反射：将基站信号反射到目标用户，支持幅度放大
    2. 干扰生成：生成人工噪声干扰窃听者
    3. 动态调度：根据场景需求分配RIS元素用途

    工作模式:
    - MR (Multi-functional RIS): 同时反射和干扰  ← 当前使用
    - AR (Active RIS): 仅反射（带放大）
    - AJ (Active Jamming): 仅干扰
    """
```

### 4.2 有源 RIS vs 无源 RIS

| 特性 | 无源 RIS (Passive) | 有源 RIS (Active) | 本模型 |
|------|-------------------|------------------|--------|
| **幅度控制** | \|Φ_nn\| = 1（纯相移） | \|Φ_nn\| ≥ 1（可放大） | **有源** ✓ |
| **信号放大** | 无，仅反射 | 有，最大放大 10 倍 | **有放大** ✓ |
| **人工噪声** | 不能生成 | 可独立生成 AN | **有 AN** ✓ |
| **功耗** | 极低（无源器件） | 较高（需要放大电路） | 有功率约束 |
| **干扰能力** | 仅被动反射 | 可主动发射干扰信号 | **主动干扰** ✓ |

### 4.6 相位映射（convert_list_to_complex_diag）

**RL agent 输出的 action 值 → RIS 复数对角元素**（`src/envs/math_tool.py`）：

```python
def convert_list_to_complex_diag(list_real, diag_row_num):
    for i in range(M):
        # 幅度映射：action ∈ [-1, 1] → amplitude ∈ [1, 10]
        amplitude = 1.0 + 9.0 * (0.5 + 0.5 * math.sin(list_real[i]))
        # 相位映射：action ∈ [-1, 1] → phase ∈ [-π, π]
        phase = list_real[i] * math.pi
        # 复数对角元素 = 幅度 × e^(j×相位)
        diag_matrix_complex[i, i] = amplitude * cmath.exp(1j * phase)
```

**映射关系图**：

```
action ∈ [-1, 1]
    │
    ├──→ amplitude = 1 + 9 × (0.5 + 0.5 × sin(action))
    │    范围: [1, 10]  ← 有源放大，最大10倍
    │
    └──→ phase = action × π
         范围: [-π, π]  ← 全相位覆盖

最终: Phi_nn = amplitude × e^(j × phase)   (复数对角元素)
```

### 4.7 RIS 相位控制（12维 → 64单元 tile 扩展）

**当前模型的核心设计**：12 维控制相位通过 `np.tile` 扩展至 64 单元

```python
# src/envs/uav_comm_env.py — step()
ris_action = Phi[:24]       # 24维: 12信号反射 + 12人工噪声
ris_half = 12

# 路径①：信号反射相位（12相位 → 64单元对角矩阵）
signal_phases = ris_action[:ris_half]
signal_expanded = np.tile(signal_phases, self.RIS.ant_num // ris_half + 1)[:self.RIS.ant_num]
self.RIS.Phi = np.asmatrix(np.diag(np.exp(1j * signal_expanded * np.pi)), dtype=complex)
self.RIS.Phi_signal = self.RIS.Phi

# 路径②：人工噪声相位（12相位 → 64单元对角矩阵）
jam_phases = ris_action[ris_half:]
jam_expanded = np.tile(jam_phases, self.RIS.ant_num // ris_half + 1)[:self.RIS.ant_num]
self.RIS.Phi_jam = np.asmatrix(np.diag(np.exp(1j * jam_expanded * np.pi)), dtype=complex)
```

**动作空间分配**：

```
Agent 1 总动作 (48维)：
├── [0:16]    BS 波束成形矩阵 G (4天线 × 2用户 × 2 = 16维)
├── [16:24]   FAS 发射矩阵 F (4端口 × 2 = 8维)
├── [24:36]   RIS 信号反射相位 Phi_signal (12单元 = 12维)
└── [36:48]   RIS 人工噪声相位 Phi_jam (12单元 = 12维)

Agent 2 总动作 (3维)：
└── [0:3]    UAV 三维移动 (Δx, Δy, Δz)
```

## 五、FAS 与 RIS 的协同工作

### 5.1 协同机制

```
FAS (无人机上)          RIS (地面固定)
    │                      │
    │  ① FAS 选择最优端口    │
    │  对准 RIS 方向发射     │
    │ ──────────────────→  │
    │                      │  ② RIS Phi_signal 反射信号
    │                      │  增强用户接收
    │                      │ ────────────→ 用户1 ✓
    │                      │ ────────────→ 用户2 ✓
    │                      │
    │                      │  ③ RIS Phi_jam 生成 AN
    │                      │  干扰窃听者
    │                      │ ═══════════→ 窃听者 ✗
```

### 5.2 优化目标

| 组件 | 优化目标 | 实现方式 |
|------|---------|---------|
| FAS | 最大化→RIS增益，最小化→窃听者增益 | 端口切换改变波束指向 |
| RIS Phi_signal | 增强用户信号（相长干涉） | 调整反射相位对齐直射路径 |
| RIS Phi_jam | 干扰窃听者（人工噪声） | 生成 AN + 相位对齐窃听者信道 |
| BS G | 波束成形，多用户干扰管理 | 空间复用 + 干扰抑制 |

### 5.3 功率分配

```
总功率 = BS发射功率 + FAS发射功率 + RIS放大功率 + AN功率

BS:     P_BS = trace(G × G^H) × power_factor    (power_factor = 100)
FAS:    P_FAS = mean(|F|)² × power_factor
RIS:    P_RIS ≤ Pr = 0 dBm = 1 mW              (总功率约束)
AN:     P_AN = P_J = 10 dBm = 10 mW            (干扰功率)
```

## 六、奖励函数设计

### 6.1 奖励组成

```python
# src/envs/uav_comm_env.py — reward()
def reward(self):
    reward = 0

    # 功率惩罚: 固定-1
    P = np.trace(self.UAV_BS_FAS.G * self.UAV_BS_FAS.G.conj().T)
    if abs(P) > abs(self.UAV_BS_FAS.G_Pmax):
        return -1.0

    # 位置奖励 [0, 10]: 距离RIS越近越好
    dist_to_ris = np.linalg.norm(uav_pos - ris_pos)
    position_reward = max(0, 10.0 * (1.0 - dist_to_ris / 50.0))
    reward += position_reward

    # 安全容量奖励 [0, 1]
    reward += min(total_secure, 1.0)

    # 用户容量奖励 [0, 1]
    reward += min(total_capacity, 1.0)

    # 窃听者惩罚 [-1, 0]
    reward -= min(total_attacker_cap, 1.0)

    return reward
```

### 6.2 奖励范围

| 组件 | 范围 | 说明 |
|------|------|------|
| 功率惩罚 | -1.0 | 超功率时立即返回 |
| 位置奖励 | [0, 10] | 距离RIS 0m→10分, 50m→0分 |
| 安全容量 | [0, 1] | 截断至1 |
| 用户容量 | [0, 1] | 截断至1 |
| 窃听惩罚 | [-1, 0] | 截断至1 |
| **总计** | **[-2, 12]** | |

## 九、注意事项

1. **FAS 端口配置 vs 实际使用**：虽然配置了 12 个端口，但实际波束成形只使用 4 个关键端口（8 维动作）。端口选择由启发式算法 `fas_beam_optimize()` 完成，不在 RL 动作空间中。

2. **RIS 相位控制**：12 维控制相位通过 `np.tile` 扩展至 64 单元，降低了 RL 动作空间维度，但牺牲了每个 RIS 单元的独立控制能力。

3. **3D 移动**：Agent 2 支持三维空间移动（包含 z 轴），可以调整无人机高度。

4. **奖励函数**：由位置奖励、容量奖励和窃听惩罚三部分组成，范围 [-2, 12]。

5. **`if_dir_link` 参数**：设为 1 时同时启用直射和 RIS 反射路径；设为 0 时仅禁用直射链路。

6. **人工噪声功率**：AN 功率固定为 -114 dBm（与接收噪声功率相同）。

7. **RIS 法向量**：当前设置为 [0, 1, 0]（指向y轴），而非 [0, 0, 1]（向上），这是为了避免与 [0, 0, 1] 平行导致 cross product 为零。
