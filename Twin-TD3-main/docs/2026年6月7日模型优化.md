```python
ris_action = Phi[:24]       # 24维: 12信号反射 + 12人工噪声
ris_half = 12

# 路径①：信号反射相位（12相位 → 64单元对角矩阵）
signal_phases = ris_action[:ris_half]
signal_expanded = np.tile(signal_phases, self.RIS.ant_num // ris_half + 1)[:self.RIS.ant_num]
self.RIS.Phi_signal = np.asmatrix(np.diag(np.exp(1j * signal_expanded * np.pi)), dtype=complex)

# 路径②：人工噪声相位（12相位 → 64单元对角矩阵）
jam_phases = ris_action[ris_half:]
jam_expanded = np.tile(jam_phases, self.RIS.ant_num // ris_half + 1)[:self.RIS.ant_num]
self.RIS.Phi_jam = np.asmatrix(np.diag(np.exp(1j * jam_expanded * np.pi)), dtype=complex)
```

```python
# src/envs/uav_comm_env.py
```

**RIS 相位矩阵体系**：

### 5.1 信道对象清单

| 信道 | 符号 | 维度 | 路径损耗指数 | 链路类型 |
|------|------|------|------------|---------|
| h_U_k[k] | $h_{d,k}$ | (1, 4) | n=3.5 | UAV→用户 直射 |
| h_U_p[p] | $h_{d,e}$ | (1, 4) | n=3.5 | UAV→窃听者 直射 |
| h_UR | $h_{UR}$ | (64, 4) | n=2.2 | UAV→RIS |
| h_R_k[k] | $h_{R,k}$ | (1, 64) | n=2.8 | RIS→用户 |
| h_R_p[p] | $h_{R,e}$ | (1, 64) | n=2.8 | RIS→窃听者 |

### 5.2 用户综合信道模型

**双路径矩阵叠加**：

```python
# src/envs/uav_comm_env.py — _ris_reflected_channel() + calculate_comprehensive_channel_of_user_k()
def _ris_reflected_channel(self, h_R, ris_phi):
    """RIS反射路径: h_R @ Phi @ h_UR.T"""
    h_R_a = np.asarray(h_R)            # (1, N_ris)
    Phi_a = np.asarray(ris_phi)        # (N_ris, N_ris)
    h_UR_a = np.asarray(self.h_UR.channel_matrix).T  # (N_ris, N_bs)
    return h_R_a @ Phi_a @ h_UR_a      # (1, N_bs)

def calculate_comprehensive_channel_of_user_k(self, k):
    h_d = np.asarray(self.h_U_k[k].channel_matrix)   # (N_bs, 1)
    fas_gain = np.mean(np.abs(self.UAV_BS_FAS.F))
    h_R_k = np.asarray(self.h_R_k[k].channel_matrix)  # (1, N_ris)
    H_reflect = self._ris_reflected_channel(h_R_k, self.RIS.Phi_signal)
    H_eff = (h_d + H_reflect.T) * fas_gain  # (N_bs, 1)
    return H_eff
```

**物理模型**：

$$H_{eff,k} = \left(h_{d,k} + h_{R,k}^H \cdot \Phi_{signal} \cdot h_{UR}\right) \cdot \overline{|F|}$$

其中：
- $h_{d,k}$：UAV→用户k 直射信道 (N_bs, 1)
- $h_{R,k}^H \cdot \Phi_{signal} \cdot h_{UR}$：UAV→RIS→用户k 级联反射信道 (1, N_bs)
- $\overline{|F|}$：FAS 流体天线平均增益（标量）

### 5.3 窃听者综合信道模型

```python
# src/envs/uav_comm_env.py — calculate_comprehensive_channel_of_attacker_p()
def calculate_comprehensive_channel_of_attacker_p(self, p):
    h_d = np.asarray(self.h_U_p[p].channel_matrix)
    fas_gain = np.mean(np.abs(self.UAV_BS_FAS.F))
    h_R_p = np.asarray(self.h_R_p[p].channel_matrix)
    H_reflect = self._ris_reflected_channel(h_R_p, self.RIS.Phi_signal)
    H_eff = (h_d + H_reflect.T) * fas_gain
    return H_eff
```

### 5.4 用户容量公式

$$R_k = \log_{10}\left(1 + \frac{|H_{eff} \cdot G_k|^2}{\|H_{eff} \cdot G_k'\|^2 + \sigma_k}\right)$$

其中：
- $H_{eff} = (h_{d,k} + h_{R,k}^H \Phi_{signal} h_{UR}) \cdot \overline{|F|}$
- $G_k$：用户k 的波束成形向量
- $G_k'$：其他用户的波束成形矩阵（多用户干扰）
- $\sigma_k$：噪声功率

### 5.5 窃听容量公式（含人工噪声）

$$R_{p,k} = \log_{10}\left(1 + \frac{|H_{eff} \cdot G_k|^2}{\|H_{eff} \cdot G_k'\|^2 + \sigma_p + P_{AN,eff}}\right)$$

其中人工噪声等效功率：

$$P_{AN,eff} = \|h_{R,p}^H \cdot \Phi_{jam}\|^2 \cdot P_{AN}$$

- $h_{R,p}^H \cdot \Phi_{jam}$：RIS→窃听者的干扰等效信道
- $P_{AN}$：人工噪声基础功率（-114 dBm）
- $\Phi_{jam}$：人工噪声相位矩阵（独立于信号反射相位）

## 六、奖励函数设计

### 6.1 奖励组成

```python
# src/envs/uav_comm_env.py — reward()
def reward(self):
    reward = 0

    # 功率惩罚: 固定-1
    P = np.trace(self.UAV_BS_FAS.G * self.UAV_BS_FAS.G.conj().T)
    if abs(P) > abs(self.UAV_BS_FAS.G_Pmax):
        return -1.0

    # 位置奖励 [0, 10]: 距离RIS越近越好
    dist_to_ris = np.linalg.norm(uav_pos - ris_pos)
    position_reward = max(0, 10.0 * (1.0 - dist_to_ris / 50.0))
    reward += position_reward

    # 安全容量奖励 [0, 1]
    reward += min(total_secure, 1.0)

    # 用户容量奖励 [0, 1]
    reward += min(total_capacity, 1.0)

    # 窃听者惩罚 [-1, 0]
    reward -= min(total_attacker_cap, 1.0)

    return reward
```

## 七、无人机能耗模型

### 7.1 旋翼无人机功率消耗模型

$$P = P_0 + P_i + P_{tip} + P_{body}$$

```python
# src/envs/uav_comm_env.py — get_energy_consumption()
def get_energy_consumption(v_t):
    # 桨尖风阻功率 + 机身阻力功率
    energy_1 = P_0 + 3 * P_0 * (abs(v_t)) ** 2 / (U_tip ** 2) + 0.5 * d_0 * rho * s * A_r * (abs(v_t))**3
    # 旋翼诱导功率 (考虑前飞修正)
    energy_2 = P_i * ((1 + (abs(v_t) ** 4) / (4 * (v_0 ** 4))) ** 0.5 - (abs(v_t) ** 2) / (2 * (v_0 **2))) ** 0.5
    # 总能耗 = 功率 × 时间
    energy = delta_time * (energy_1 + energy_2)
    return energy
```

## 十一、注意事项

1. **动作空间维度变化**：当前 Agent 1 动作空间为 48 维（16 BS + 8 FAS + 24 RIS），Agent 2 为 3 维。与之前的 192 维设计完全不同。

2. **状态空间维度**：Agent 1 为 42 维，Agent 2 为 9 维。

3. **RIS 相位控制**：12 维控制相位通过 `np.tile` 扩展至 64 单元，降低了 RL 动作空间维度，但牺牲了每个 RIS 单元的独立控制能力。

4. **FAS 端口使用**：虽然配置了 12 个端口，但实际波束成形只使用 4 个关键端口（8 维动作），端口选择由启发式算法 `fas_beam_optimize()` 完成。

5. **3D 移动**：Agent 2 支持三维空间移动（包含 z 轴），这是对旧版 2D 移动的扩展。

6. **奖励函数简化**：当前奖励函数由位置奖励、容量奖励和窃听惩罚三部分组成，范围 [-2, 12]，比旧版更简洁直观。

7. **`if_dir_link` 参数**：设为 1 时同时启用直射和 RIS 反射路径；设为 0 时仅禁用直射链路，RIS 反射链路仍通过 UAV→RIS 工作。

8. **已训练模型兼容性**：由于动作空间和状态空间维度变化，不同版本的模型不能互相加载。
