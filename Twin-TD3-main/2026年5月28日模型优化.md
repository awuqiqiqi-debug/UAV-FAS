# 模型优化记录 - 2026年5月28日

## 优化概述

本次优化针对 UAV-BS-FAS 保密通信系统的信道模型，使其与实际物理模型完全对齐。主要修复了 RIS 信道建模不完整、信号路径缺失、人工噪声未显式建模等问题。

## 二、详细优化说明

### 2.1 新增 RIS 反射链路信道

**优化前**：系统仅初始化了直射链路信道 `h_U_k`（UAV→用户）和 `h_U_p`（UAV→窃听者），缺少 RIS 相关的级联信道。

**优化后**：新增三个信道对象，覆盖完整的 RIS 反射链路：

```python
# UAV → RIS 信道 (ris_ant_num, bs_ant_num) = (16, 8)
self.h_UR = mmWave_channel(self.RIS, self.UAV_BS_FAS, fre)

# RIS → 用户k 信道 (1, ris_ant_num) = (1, 16)
self.h_R_k = [mmWave_channel(self.RIS, user_k, fre) for user_k in self.user_list]

# RIS → 攻击者p 信道 (1, ris_ant_num) = (1, 16)
self.h_R_p = [mmWave_channel(self.RIS, attacker_p, fre) for attacker_p in self.attacker_list]
```

**物理意义**：RIS 位于地面固定位置 (12.5, 12.5, 0)，信号从 UAV 发射后经 RIS 反射到达用户/窃听者。三条链路各自有独立的路径损耗和信道矩阵。

---

### 2.2 用户综合信道模型

**优化前**（标量平均近似）：
```python
enhance_factor = np.mean(np.exp(1j * enhance_phase)) * (1 + 0.8 * np.mean(np.abs(Psi_user_diag)))
H_eff = h_U_k.H * enhance_factor * fas_gain
```
问题：16个RIS单元的相位被压缩成一个标量，丢失了空间维度信息。

**优化后**（双路径矩阵叠加）：
```python
# 直射路径
H_direct = h_d.H                              # (bs_ant_num, 1)

# RIS反射路径
H_reflect = h_R_k.H * Phi_signal * h_UR       # (bs_ant_num, 1)

# 综合信道 = 直射 + RIS反射
H_eff = (H_direct + H_reflect) * fas_gain
```

**物理模型**：
$$H_{eff,k} = \left(h_{d,k}^H + h_{R,k}^H \cdot \Phi_{signal} \cdot h_{UR}\right) \cdot \overline{|F|}$$

其中：
- $h_{d,k}^H$：UAV→用户k 直射信道
- $h_{R,k}^H \cdot \Phi_{signal} \cdot h_{UR}$：UAV→RIS→用户k 级联反射信道
- $\overline{|F|}$：FAS 流体天线平均增益

---

### 2.3 窃听者综合信道模型

**优化前**：使用 `interference_factor` 标量近似，且 `calculate_comprehensive_channel_of_attacker_p()` 中有 `fas_gain` 但 `calculate_capacity_array_of_attacker_p()` 中遗漏。

**优化后**：与用户信道统一采用双路径模型：
```python
H_direct = h_d.H
H_reflect = h_R_p.H * Phi_signal * h_UR
H_eff = (H_direct + H_reflect) * fas_gain
```

---

### 2.4 用户容量公式

**优化后**：
$$R_k = \log_{10}\left(1 + \frac{|H_{eff} \cdot G_k|^2}{\|H_{eff} \cdot G_k'\|^2 + \sigma_k}\right)$$

其中 $H_{eff} = (h_{d,k}^H + h_{R,k}^H \Phi_{signal} h_{UR}) \cdot \overline{|F|}$

---

### 2.5 窃听容量公式（含人工噪声）

**优化前**：缺少 `fas_gain`，无 AN 项。

**优化后**：
$$R_{p,k} = \log_{10}\left(1 + \frac{|H_{eff} \cdot G_k|^2}{\|H_{eff} \cdot G_k'\|^2 + \sigma_p + P_{AN,eff}}\right)$$

其中人工噪声等效功率：
$$P_{AN,eff} = \|h_{R,p}^H \cdot \Phi_{jam}\|^2 \cdot P_{AN}$$

- $h_{R,p}^H \cdot \Phi_{jam}$：RIS→窃听者的干扰等效信道
- $P_{AN}$：人工噪声基础功率（-114 dBm）
- $\Phi_{jam}$：人工噪声相位矩阵（独立于信号反射相位）

---

### 2.6 人工噪声(AN)显式建模

**优化前**：无显式 AN，仅通过 `interference_factor` 近似。

**优化后**：在 `step()` 中生成 AN 向量：
```python
self.an_power = dB_to_normal(-114) * 1e-3  # 基础噪声功率(mW)
self.z_AN = (np.random.randn(N, 1) + 1j * np.random.randn(N, 1)) / np.sqrt(2)
```

窃听者接收信号模型：
$$y_e = \underbrace{(h_d^H + h_{R,p}^H \Phi_{signal} h_{UR}) \sqrt{P} w_k s_k}_{信号} + \underbrace{h_{R,p}^H \Phi_{jam} \mathbf{z}_{AN}}_{人工噪声} + n_e$$

---

### 2.7 RIS 相位矩阵分离

**优化前**：
```python
self.RIS.Phi_user = self.RIS.Phi       # 同一个矩阵
self.RIS.Phi_attacker = self.RIS.Phi   # 同一个矩阵
```

**优化后**：Phi 的后 32 维拆分为两个独立矩阵：
```python
ris_action = Phi[fas_ant_num*2:]       # RIS 的 32 维动作
# 前16维 → 信号反射相位矩阵（增强用户信号）
self.RIS.Phi_signal = convert_list_to_complex_diag(ris_action[:16], 16)
# 后16维 → 人工噪声相位矩阵（干扰窃听者）
self.RIS.Phi_jam = convert_list_to_complex_diag(ris_action[16:], 16)
```

---

### 2.8 CSI 更新范围

**优化前**：仅更新直射链路 CSI。
```python
for h in self.h_U_k + self.h_U_p:
    h.update_CSI()
```

**优化后**：更新全部链路 CSI（直射 + RIS 反射）。
```python
for h in self.h_U_k + self.h_U_p + [self.h_UR] + self.h_R_k + self.h_R_p:
    h.update_CSI()
```

---

### 2.9 动作空间映射

动作空间维度不变（Agent 1: 96维，Agent 2: 2维），但 Phi 参数的内部映射改变：

| 参数范围 | 维度 | 含义 |
|---------|------|------|
| action_1[0 : 2×bs_ant_num×user_num] | 32 | BS 波束成形矩阵 G |
| action_1[32 : 32+2×fas_ant_num] | 32 | FAS 流体天线发射矩阵 F |
| action_1[64 : 64+ris_ant_num] | 16 | RIS 信号反射相位 (Phi_signal) |
| action_1[80 : 80+ris_ant_num] | 16 | RIS 人工噪声相位 (Phi_jam) |
| action_2[0:2] | 2 | UAV 移动 (Δx, Δy) |

## 三、模型对比

### 信号路径对比

| 路径 | 优化前 | 优化后 |
|------|--------|--------|
| UAV→用户 (直射) | 被标量因子替代 | 独立直射路径 h_d^H |
| UAV→RIS→用户 (反射) | 标量平均近似 | 矩阵级联 h_R_k^H Φ_signal h_UR |
| UAV→窃听者 (直射) | 被标量因子替代 | 独立直射路径 h_d^H |
| UAV→RIS→窃听者 (反射) | 标量平均近似 | 矩阵级联 h_R_p^H Φ_signal h_UR |
| RIS→窃听者 (AN) | 无 | h_R_p^H Φ_jam z_AN |

### 接收信号对比

**优化前（用户k）**：
$$y_k = h_{U_k}^H \cdot \overline{\Psi} \cdot (1 + 0.8\overline{|\Psi|}) \cdot \overline{|F|} \cdot \sqrt{P} w_k s_k + n_k$$

**优化后（用户k）**：
$$y_k = \left(h_{d,k}^H + h_{R,k}^H \Phi_{signal} h_{UR}\right) \cdot \overline{|F|} \cdot \sqrt{P} w_k s_k + n_k$$

**优化后（窃听者e）**：
$$y_e = \left(h_{d,e}^H + h_{R,e}^H \Phi_{signal} h_{UR}\right) \cdot \overline{|F|} \cdot \sqrt{P} w_k s_k + h_{R,e}^H \Phi_{jam} \mathbf{z}_{AN} + n_e$$

## 五、阵列间距修正

### 5.1 问题

优化前 FAS 端口间距为 0.05m（5cm），不符合 28GHz 毫米波的 0.5λ 标准。

### 5.2 修正计算

```
28GHz 波长:  λ = c/f = 3×10⁸ / 28×10⁹ ≈ 1.071 cm
标准间距:    d = 0.5λ ≈ 0.536 cm = 0.00536 m
FAS 总长度:  (16-1) × 0.00536 ≈ 8.04 cm（无人机机身可容纳）
```

### 5.3 代码修改

**`entity.py`** — UAV_BS_FAS 类：
```python
# 修改前
self.fas_port_local = np.array([
    [0.05 * i, 0.0, 0.0] for i in range(-fas_num_ports // 2, fas_num_ports // 2)
])

# 修改后
self.fas_port_spacing = 0.5 * 3e8 / 28e9  # 0.5λ = 0.00536m
self.fas_port_local = np.array([
    [self.fas_port_spacing * i, 0.0, 0.0] for i in range(-fas_num_ports // 2, fas_num_ports // 2)
])
```

**`FASModule.py`** — UAVMountedFAS 类：
```python
# 修改前
self.port_local = np.array([
    [0.05 * i, 0.0, 0.0] for i in range(-self.num_ports // 2, self.num_ports // 2)
])

# 修改后
self.port_spacing = 0.5 * 3e8 / 28e9  # 0.5λ = 0.00536m
self.port_local = np.array([
    [self.port_spacing * i, 0.0, 0.0] for i in range(-self.num_ports // 2, self.num_ports // 2)
])
```

**`entity.py`** — FAS 天线类型修正：
```python
# 修改前：fas_ant_type='UPA'（面阵，与实际线性排布不符）
# 修改后：fas_ant_type='ULA'（线阵，与物理排布一致）
```

|------|------|--------|------|--------|

## 六、RIS 单元数量优化

### 6.1 优化背景

原代码中 RIS 采用 4×4=16 个反射单元，物理尺寸仅约 1.61cm×1.61cm，尺寸过小，不利于实际部署和信号反射效果。

### 6.2 优化方案

将 RIS 从 16 单元 (4×4 UPA) 扩展为 64 单元 (8×8 UPA)，物理尺寸增大至约 3.75cm×3.75cm。

**优化前**：
- RIS 单元数：16 (4×4 UPA)
- 物理尺寸：≈1.61cm × 1.61cm
- 信道维度：h_UR (16, 8), h_R_k (1, 16), h_R_p (1, 16)

**优化后**：
- RIS 单元数：64 (8×8 UPA)
- 物理尺寸：≈3.75cm × 3.75cm
- 信道维度：h_UR (64, 8), h_R_k (1, 64), h_R_p (1, 64)

### 6.3 代码修改

**`env_uav_bs_fas.py`** — 默认参数修改：
```python
# 修改前
ris_ant_num = 16

# 修改后
ris_ant_num = 64
```

**`main_train_uav_bs_fas.py`** — 训练脚本修改：
```python
# 修改前
system = MiniSystem(
    ...
    ris_ant_num=16,     # RIS反射单元数（地面固定，4x4 UPA）
    ...
)

# 修改后
system = MiniSystem(
    ...
    ris_ant_num=64,     # RIS反射单元数（地面固定，8x8 UPA）
    ...
)
```

### 6.4 影响分析

1. **信道矩阵维度自动适配**：h_UR, h_R_k, h_R_p 信道对象会自动根据 ris_ant_num 调整维度，无需手动修改。

2. **相位矩阵维度自动适配**：RIS.Phi, Phi_signal, Phi_jam 等对角矩阵维度自动从 16×16 变为 64×64。

3. **动作空间维度影响**：
   - Agent 1 动作空间：原 96 维 (32 BS + 32 FAS + 16 Phi_signal + 16 Phi_jam)
   - 优化后：192 维 (32 BS + 32 FAS + 64 Phi_signal + 64 Phi_jam)
   - **需要调整 Agent 1 网络输出层维度**

4. **状态空间维度影响**：
   - 综合信道维度：原 2×(K+P)×N_bs + 3 = 51
   - 优化后：2×(K+P)×N_bs + 3 = 51（维度不变，因为信道已压缩为向量）

5. **物理尺寸合理性**：64 单元 8×8 UPA 总尺寸约 3.75cm×3.75cm，适合地面固定部署，且能提供更窄的波束和更高的反射增益。

### 6.5 三类阵列最终尺寸汇总

| 阵列 | 类型 | 阵元数 | 间距 | 总尺寸 | 位置 |
|------|------|--------|------|--------|------|
| BS | ULA 线阵 | 8 | 0.5λ ≈ 0.536cm | (8-1)×0.536 ≈ 3.75cm | 无人机上 |
| FAS | ULA 线阵 | 16 | 0.5λ ≈ 0.536cm | (16-1)×0.536 ≈ 8.04cm | 无人机上 |
| RIS | UPA 面阵 | 8×8=64 | 0.5λ ≈ 0.536cm | 7×0.536 ≈ 3.75cm × 3.75cm | 地面固定 |

## 七、注意事项

1. **动作空间维度变化**：由于 RIS 从 16 单元扩展到 64 单元，Agent 1 的动作空间维度从 96 维增加到 192 维（32 BS + 32 FAS + 64 Phi_signal + 64 Phi_jam）。需要调整 Agent 1 网络输出层维度。

2. **状态空间维度不变**：综合信道维度仍为 2×(K+P)×N_bs + 3 = 51，但综合信道的计算已更新为双路径模型。

3. **`if_dir_link` 参数**：设为 1 时同时启用直射和 RIS 反射路径；设为 0 时仅禁用直射链路（UAV→用户/窃听者），RIS 反射链路仍通过 UAV→RIS 工作。

4. **人工噪声功率**：当前 AN 功率固定为 -114 dBm（与接收噪声功率相同）。如需调整 AN 功率，修改 `step()` 中的 `self.an_power` 即可。

5. **RIS 尺寸变化**：RIS 从 4×4=16 单元 (≈1.61cm×1.61cm) 扩展为 8×8=64 单元 (≈3.75cm×3.75cm)，更适合地面固定部署，能提供更窄的波束和更高的反射增益。

6. **已训练模型不兼容**：由于动作空间维度变化（96→192），已训练的模型需要重新训练，不能直接加载旧模型继续训练。
